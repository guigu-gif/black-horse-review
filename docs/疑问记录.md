## 🔧 Java工具和环境

### 2026-02-16
**Q1: 什么是头文件？Java有头文件吗？**
- **疑问类型：** 概念混淆
- **答案：** Java没有头文件。头文件（.h）是C/C++的概念，用于声明函数和类。Java的import语句只是引用其他包的类，不是头文件。
- **状态：** ✅ 已解决

**Q2: 源文件是什么？编译后变成什么？**
- **疑问类型：** 基础概念
- **答案：**
  - 源文件：.java文件，人类可读的代码
  - 编译后：.class文件（字节码），JVM可执行的指令
  - 存放位置：target/classes/目录
- **状态：** ✅ 已解决

**Q3: 项目里的依赖是从哪来的？**
- **疑问类型：** 工具使用
- **答案：**
  - Maven从中央仓库（https://repo.maven.apache.org/）下载
  - 下载后存储在本地：C:\Users\用户名\.m2\repository\
  - pom.xml里配置需要哪些依赖
- **状态：** ✅ 已解决

**Q4: 怎么看报错信息？编译错误和运行错误有什么区别？**
- **疑问类型：** 调试技能
- **答案：**
  - 编译错误：代码还没运行，语法写错了（红色波浪线）
  - 运行错误：代码运行时出问题了（Exception异常）
  - 堆栈信息：从下往上看，找到自己写的代码行
- **状态：** ⏳ 待实践验证

**Q13: 启动项目后报错"Public Key Retrieval is not allowed"**
- **疑问类型：** 数据库连接问题
- **答案：**
  - 根本原因：MySQL 8.0使用`caching_sha2_password`认证，但项目的MySQL驱动是5.1.47（老版本），不支持新认证方式
  - 解决方案：在数据库连接URL中添加`&allowPublicKeyRetrieval=true`参数
  - 修改位置：`application.yaml` 第8行
- **状态：** ✅ 已解决

## 🔧 Java工具和环境

### 2026-02-16
**Q5: 为什么IDEA启动项目后，前端页面显示"店铺加载失败"？**
- **疑问类型：** 项目启动问题
- **答案：**
  - 根本原因：IDEA每次运行"Start Nginx"都执行 `start nginx.exe`，不检查是否已运行
  - 导致结果：每次启动创建2个新nginx进程（1 master + 1 worker）
  - 积累效果：多次启动后有10+个nginx进程监听8080端口
  - 端口冲突：多个nginx争抢请求，导致前端加载失败
  - 解决方案：
    1. 创建 `start-nginx-safe.bat` 脚本，启动前先检查是否已运行
    2. 修改IDEA的Start_Nginx.xml配置，调用安全脚本
    3. 创建 `stop-nginx.bat` 优雅停止脚本
    4. 清理所有冗余nginx进程
- **状态：** ✅ 已解决
